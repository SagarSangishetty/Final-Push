Why Helm via Terraform and not manual helm install:

Manual helm install → not tracked in state → not reproducible → drift possible

Helm via Terraform → declared as code → single apply → state tracks chart versions and valuesSeparate state file:

Why exec and not static token → EKS tokens expire every 15 minutes → exec fetches fresh token on every call → long applies never fail with expired token3 Helm Releases:LB Controller

Chart → aws-load-balancer-controller
Repo → https://aws.github.io/eks-charts
Namespace → kube-system
Key values → clusterName, replicaCount=2
IRSA → serviceAccount.annotations.eks.amazonaws.com/role-arn = lb_controller_role_arn from IAM remote state
Purpose → watches Ingress resources, creates ALBs in AWS automatically
Cluster Autoscaler

Chart → cluster-autoscaler
Repo → https://kubernetes.github.io/autoscaler
Namespace → kube-system
Key values → autoDiscovery.clusterName, awsRegion
IRSA → rbac.serviceAccount.annotations.eks.amazonaws.com/role-arn = cluster_autoscaler_role_arn from IAM remote state
Purpose → scales node group up when pods are pending, scales down after 10min underutilisation
External Secrets Operator

Chart → external-secrets
Repo → https://charts.external-secrets.io
Namespace → external-secrets (created via kubernetes_namespace resource first)
Key values → installCRDs=true
IRSA → serviceAccount.annotations.eks.amazonaws.com/role-arn = external_secrets_role_arn from IAM remote state
Purpose → watches ExternalSecret CRDs, fetches value from Secrets Manager, creates K8s Secret automatically
