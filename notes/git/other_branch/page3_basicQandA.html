Q1. What does git status show?
...shows the current status of this file whether it is tracked untracked or we have any file to commit or working tree is clean
Q2. What is the purpose of git add?
...to add the file is staging area
Q3. What does git commit -m "msg" do?
...it will commit files which are in staging area with a commit message.
Q4. Difference between git commit and git push?
...git commit will commit the files on a local repo while git push will push the commited file to remote repo.
Q5. What does git clone do?
...it will clone the entire remote repo to your local machine.
Q6. What is the meaning of --global in:
...global here is setting email for all the repo present on the particular system.
Q7. Which command shows where Git is getting the email from?
...git config --show-origin user.email
Q8. What is the difference between --system, --global, and --local config?
...system = git settings for all user and all repo.
...global = git settings for particular os user and all repo
...local = for one particular repo for a user on a system
Q9. Remove a wrong global email setting. Command?
...git --config --global --unset user.email
Q10. What does git log display?
...It will show summary of commit history for files example like date time user name email commit message etc
Q11. What does git log -p -2 show?
shows last 2 commits with actual code diff (patch)
Q12. What is the use of: git log -S "password"
Finds commits where "password" was added or removed.
Q13. Which command shows commit summary only (not full diff)?
...git log --stat
Q14. What does git diff show?
...shows the changes which are made.
Q15. What does git show <commit-id> do?
...shows the changes made for that particular commit id
Q16. What is the purpose of: git checkout branch-name
...to switch branches.
Q17. What does this do? git checkout <commit-id> -- file.txt
...it will take file to that particular commit id which means file will have all the changes for that commit id.
Q18. Difference between: git restore file.txt & git restore --staged file.txt
...git restore file.txt -- will undo the changes made in the file which is not save i.e not staged.

...git restore --staged file.txt -- it will also undo the changes made in which after adding the file in staged area.

git restore file.txt ‚Üí undo working directory changes
git restore --staged file.txt ‚Üí unstage but keep changes
Q19. Difference between git reset --soft and git reset --hard?
...git reset --soft means file undo the changes but will not delete the changes which are in file & git reset --hard will undo the changes and delete the changes as well
Q20. What does git remote -v show?
...It will show the remote url of the git hub repo
